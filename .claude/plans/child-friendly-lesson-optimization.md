# 儿童化引导与逐课内容优化计划

## 目标

只优化用户明确认可的方向：

1. 让孩子在“今日课”里看到更儿童化、更有角色感的学习引导。
2. 针对 28 节课逐节补充更清晰的学习口号、任务身份、关键提醒和内容微调。
3. 不把项目改成通用发布版；不强行同步所有文档；不改用户自用所需的整体结构。

## 不做

- 不修改课程总周期、周数、节数定位。
- 不重写 `COURSE_DESIGN.md` 等文档。
- 不调整导航、功能模块和产品定位。
- 不删除现有家长/陪学内容。
- 不做大规模视觉重设计。

## 代码现状

- 课程基础数据在 `app.js`：`lessonSeeds` 和 `lessonEnrichments`。
- 每节课通过 `course-upgrade.js` 的 `enrichLessons()` 生成 `lesson.upgraded`。
- “今日课”主要由 `app.js` 的 `renderLesson()`、`renderStepPanel()`、`renderSelfStudyCard()`、`renderStuckHints()`、`renderSelfCheck()` 渲染。
- 目前增强数据偏方法化，例如“我能独立选择数位图”，对家长友好，但对孩子可以更像闯关任务。

## 实施方案

### 1. 在 `course-upgrade.js` 增加逐课儿童化配置

新增一个按课程序号索引的配置表，例如 `childLessonGuides`，覆盖 28 节课。

每节课包含：

- `role`：孩子的任务身份，例如“小票侦探”“点阵摆盘师”“错因侦探”“分数切分师”。
- `slogan`：一句孩子能记住的话，例如“数位先站队，算完倒着查”。
- `mission`：今天的闯关任务，例如“帮小票找总价，再倒着检查答案”。
- `watch`：本节最需要盯住的点，例如“不要把所有错误都叫粗心”。
- `tryThis`：可选的小挑战或表达任务，例如“把一道错题做成错因卡”。

这部分不替代原课程内容，而是叠加在 `upgraded.childGuide` 中。

### 2. 调整 `course-upgrade.js` 的生成逻辑

在 `enrichLesson()` 里增加：

```js
childGuide: buildChildGuide(lesson)
```

同时轻微优化：

- `buildMeta()` 的输出可以优先使用儿童化 `mission`，但保留原能力线 mission。
- `buildSelfCheck()` 可以加入更孩子化的检查项，如“我能当小老师讲一遍”。
- `buildPracticeSets()` 的 `reasoningPrompt` 可以从统一的“先说图，再说算式”扩展为本课专属提示。

### 3. 在 `app.js` 的今日课界面展示儿童化引导

修改以下渲染函数：

- `renderStepPanel()`：在当前任务上方或任务板里显示角色和口号。
- `renderSelfStudyCard()`：显示“今天我是：xxx”和一句闯关任务。
- `renderStuckHints()`：加入本课 `watch`，让孩子知道最容易掉坑的地方。
- `renderSelfCheck()`：加入 `tryThis`，作为复盘后的可选挑战。

展示方式保持现有样式体系，不做大 UI 改造。必要时在 `styles.css` 增加少量样式，例如 `.child-role-card`、`.kid-slogan`。

### 4. 逐课优化内容方向

28 节课的优化重点如下：

1. 加减竖式与验算：强调“数位站队”和“倒着查”。
2. 乘法口诀与乘除互逆：强调“同一张点阵讲三道算式”。
3. 两步混合运算顺序：强调“先给符号排队”。
4. 计算错因侦探课：强调“不说粗心，说证据”。
5. 加法凑整：强调“先找好朋友”。
6. 减法补偿：强调“多走退回，少走补上”。
7. 乘法拆分：强调“把大块切成好算的小块”。
8. 策略选择课：强调“不硬套，先观察”。
9. 条件圈画：强调“先收拾题目，再列式”。
10. 重叠问题：强调“中间被数了几次”。
11. 图形拼剪：强调“面积看小格，周长看外边”。
12. 方位观察：强调“眼睛站在哪里”。
13. 整体1与平均分：强调“先问整体是谁”。
14. 几分之一：强调“整体一样，分得越多每份越小”。
15. 几分之几：强调“下面分几份，上面取几份”。
16. 生活中的分数：强调“生活分数也要说整体”。
17. 读表格：强调“标题、行列、单位”。
18. 条形图比较：强调“读刻度，不只看长短”。
19. 线段图解应用题：强调“谁多画长，问题标上”。
20. 找规律：强调“先圈一组，再找位置”。
21. 计算与速算闯关：强调“先判断关卡类型”。
22. 逻辑空间综合题：强调“先选图，不凭感觉猜”。
23. 分数与图表综合题：强调“总量要一致”。
24. 家庭数学游园会项目：强调“能教别人，才是真的会”。
25. 画图整理条件：强调“按事情顺序接箭头”。
26. 阵列图和分组图：强调“几个几、怎么分”。
27. 线段图和倍数图：强调“多几看差，几倍看段”。
28. 画图思维挑战：强调“一题多图，同一个答案”。

### 5. 验证

完成后执行：

```bash
node --check app.js
node --check course-upgrade.js
```

如果可用，再用浏览器预览检查：

- 今日课是否显示角色/口号/任务。
- 切换几节课后儿童化内容是否随课程变化。
- 现有口算、分数、空间、画图、错题、打印模块不受影响。

## 风险与控制

- 风险：新增文字太多导致今日课拥挤。
  - 控制：只在侧栏和当前任务板放短句，不插入大段说明。
- 风险：儿童化表达过幼稚。
  - 控制：使用“小侦探/小老师/设计师/闯关”等二升三仍可接受的表达，避免过度卡通化。
- 风险：改动影响已有课程生成。
  - 控制：新增字段全部有 fallback；渲染函数读取不到 `childGuide` 时仍使用原内容。
