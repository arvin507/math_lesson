## 2. 开发目标

开发目标是把课程设计落地成一个本地可打开的互动课程网页。

优先级：

1. 先让24节课程完整可读。
2. 再让关键图示和互动练习可用。
3. 最后完善错题、进度和打印体验。

不使用后端，不依赖网络，不引入外部包，保证直接打开 index.html 即可使用。

## 3. 主要实现内容

### 3.1 新增 app.js

app.js 负责所有课程数据、渲染逻辑和交互逻辑。

需要包含：

- abilities：五大能力线数据。
- weeks 或 lessons：6周24节课程数据。
- state：当前课程、完成状态、错题记录。
- renderOverview()：渲染课程总览。
- renderLesson()：渲染单节课程。
- renderMentalPractice()：渲染口算练习。
- renderFractionLab()：渲染分数实验台。
- renderOverlapPractice()：渲染重叠格子练习。
- renderMistakes()：渲染错题本。
- renderPrintSheet()：渲染每周打印单。
- bindEvents()：绑定所有按钮、选择框、表单事件。

### 3.2 课程数据结构

每节课建议包含：

- id
- week
- number
- title
- ability
- goal
- parentScript
- visual
- method
- questions
- warmups
- exercises
- mistakeTips
- reviewPrompt

每道练习建议包含：

- level：基础、核心、挑战
- question
- answer
- explanation

### 3.3 图示实现

优先使用 HTML/CSS 图示，不依赖图片资源。

需要实现的图示类型：

- 数位表
- 数轴跳格图
- 点阵图
- 长方形拆分图
- 维恩图或重叠格子图
- 分数条
- 圆形分数图
- 条形统计图
- 线段图
- 图形规律格子图

### 3.4 互动功能

需要实现：

- 顶部导航切换不同视图。
- 总览点击课程进入详情页。
- 标记课程完成并保存到 localStorage。
- “继续下一课”跳转到未完成的第一节课。
- 口算按题型生成12题并检查答案。
- 分数实验台切换不同分数并更新图示。
- 重叠格子输入答案后检查并高亮重叠区域。
- 错题本新增、显示、删除记录。
- 打印页按周生成练习单并调用浏览器打印。

6. 实现分数实验台。

9. 实现打印练习单。

## 5. 验收标准

必须满足：

- 打开 index.html 后页面无明显空白模块。
- 总览能看到5条能力线、6周安排、24节课程。
- 每节课都有目标、讲解、图示、方法、练习、答案解析、复盘。
- 口算练习可以生成、输入、检查。
- 分数实验台可以动态切换图示。
- 重叠练习可以检查答案并高亮。
- 错题记录刷新后仍保留。
- 打印页能按周显示练习单。
- 手机宽度下文本不溢出、不重叠。
- 打印时隐藏导航，只显示练习单。

## 6. 明确暂不做

第一版暂不做：

- 后端账号系统。
- 云端同步。
- AI自动批改。
- 复杂动画。
- 外部图片素材。
- 多学生管理。
- 教师端管理后台。

## 7. 中断恢复提示

如果开发中断，恢复时先检查：

1. 是否存在 app.js。
2. index.html 是否仍引用 app.js。
3. 浏览器控制台是否有 JavaScript 错误。
4. 总览页是否能渲染24节课程。
5. localStorage 是否保存完成状态和错题。
6. 打印页是否能生成当前选择周的练习单。

当前最新决策：

- 输出形态：互动网页。
- 课时：6周24节。
- 难度：基础巩固 + 轻拔高。
- 使用者：家长带孩子。
- 图文风格：清晰数学图示。
- 开发优先级：先课程内容，后互动功能。
